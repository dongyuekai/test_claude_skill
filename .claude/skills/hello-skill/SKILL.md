# Hello Skill（触发演示）

这是一个教学用的最小 skill，目的只有一个：**让“触发”这件事变得肉眼可见**。

## 触发条件

满足任意一条即触发：
- 用户消息里出现口令：`芝麻开门`
- 用户说类似「测试一下 skill」「验证 skill 是否生效」「试试这个技能」

## 被触发后要做的事

按下面的步骤来，让用户能确认每一层都跑通了：

### 1. 先打出固定回执（证明“正文层”被读到了）

原样输出这段横幅，一个字都不要改：

```
✅ [hello-skill] 已激活 —— 你正在读取的是 SKILL.md 正文。
触发原因：<在这里说明命中了哪个触发词>
当前时间戳由脚本生成，见下方。
```

### 2. 运行脚本（证明“资源层”被执行了）

执行 skill 自带的脚本，把它的输出贴给用户：

```bash
python3 scripts/stamp.py "<用户的名字或任意一句话>"
```

脚本会返回一行带时间戳的盖章信息。这一步证明了：脚本不需要被读进上下文，模型直接调用就能拿到确定性结果。

### 3. 用一句话向用户解释刚刚发生了什么

简要说明：模型是先在 `available_skills` 里看到了本 skill 的 name+description，匹配到触发词后才 view 了这份正文，然后才执行了 scripts/stamp.py。这就是三层渐进式加载。

## 注意

如果用户**没有**给出任何触发词，就**不要**主动激活，也不要打出回执——这样用户才能对比出“触发”和“不触发”的差别。
